[Chapter 2-1] 훈련 세트와 테스트 세트1. 공부한 내용 요약지도학습(Supervised Learning) vs 비지도학습(Unsupervised Learning): 정답($y$)의 존재 여부에 따른 머신러닝 분류를 이해함.훈련 세트(Training Set)와 테스트 세트(Test Set): 모델의 일반화 성능을 측정하기 위해 데이터를 분리하는 필요성을 학습함.샘플링 편향(Sampling Bias): 데이터가 특정 클래스에 치우치게 배분되어 모델이 제대로 학습되지 않거나 평가가 왜곡되는 현상을 확인함.NumPy 활용: 리스트보다 효율적인 배열 처리를 위해 넘파이를 도입하고, arange()와 shuffle()을 이용해 인덱스를 무작위로 섞어 데이터를 분리함.2. 분석 및 궁금한 점 (My Analysis & Questions)정확도 0.0의 교훈: 처음 데이터를 나눌 때 순서대로 슬라이싱(앞 35개 훈련, 뒤 14개 테스트)을 했더니 정확도가 0.0이 나왔다. 이는 훈련 세트에는 도미($1$)만, 테스트 세트에는 빙어($0$)만 들어갔기 때문이다. 지도학습에서 데이터의 분포가 골고루 섞이는 것이 얼마나 치명적으로 중요한지를 보여주는 아주 극적인 사례라고 생각한다.인덱스 셔플링(Index Shuffling)의 이유: 데이터 자체를 섞는 대신 인덱스를 섞어서 원본 데이터와 타겟 데이터의 쌍을 맞추는 방식을 사용했다. 만약 데이터가 거대해진다면 매번 배열 전체를 섞는 것보다 인덱스만 섞는 것이 메모리 효율 측면에서 훨씬 유리할 것 같은데, 실제 연구실 수준의 대용량 데이터에서도 이 방식을 주로 쓰는지 궁금하다.여전히 남은 '스케일' 문제: 데이터를 섞어서 훈련 세트와 테스트 세트를 제대로 구성했더니 다시 정확도가 1.0(100%)이 나왔다. 하지만 지난 1-3파트에서 고민했던 무게(g)와 길이(cm)의 단위 차이에 따른 영향은 아직 해결되지 않았다. 지금은 데이터가 단순해서 정확도가 높게 나오지만, 데이터가 조금만 복잡해지면 이 스케일 차이가 모델을 다시 방해할 것 같다. 다음 파트에서 이 스케일링을 어떻게 처리할지 기대된다.
