# [Chapter 2-1] 훈련 세트와 테스트 세트

### 1. 공부한 내용 요약
- **지도학습과 비지도학습**: 정답($y$)이 있는 데이터를 통해 상관관계를 찾는 지도학습과, 정답 없이 데이터 자체의 패턴을 찾는 비지도학습의 차이를 이해했다.
- **훈련 세트와 테스트 세트**: 모델의 일반화 성능을 정직하게 측정하기 위해 전체 데이터를 학습용(35개)과 검증용(14개)으로 분리했다.
- **샘플링 편향(Sampling Bias)**: 무작위로 섞지 않고 순서대로 데이터를 나눌 경우, 특정 클래스가 한쪽에 몰려 모델 평가가 왜곡되는 현상을 확인했다.
- **NumPy 활용**: 파이썬 리스트를 넘파이 배열로 변환하고, 인덱스를 무작위로 섞어(np.random.shuffle) 편향 없는 데이터 세트를 구축했다.


### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **정확도 0.0이 준 교훈**: 처음 데이터를 나눌 때 단순히 슬라이싱만 했더니 점수가 0.0이 나왔다. 이는 모델의 결함이 아니라 데이터 배분의 문제였는데, 도미만 공부하고 빙어 시험을 치게 한 셈이라 통계학 전공자로서 데이터 분포의 중요성을 몸소 체험했다.
- **재현성을 위한 시드 고정**: 무작위 섞기를 할 때 np.random.seed(42)를 설정하는 법을 배웠다. 통계 분석에서 실험 결과의 재현은 필수적이므로, 이러한 랜덤성 제어 습관을 실무에서도 잘 챙겨야겠다고 생각했다.

- **인덱스 셔플링의 효율성**: 데이터 자체를 물리적으로 섞는 대신 인덱스 배열을 만들어 섞는 방식을 사용했다. 데이터가 매우 방대해질 경우 전체 데이터를 섞는 것보다 인덱스만 조절하는 게 메모리 측면에서 훨씬 유리할 것 같은데, 실제 대규모 데이터 처리 현장에서도 이 방식이 표준인지 궁금하다.
- **여전한 스케일 문제**: 데이터를 잘 섞어서 정확도는 다시 $1.0$이 나왔지만, 산점도를 그려보니 여전히 **x축(길이)과 y축(무게)의 단위 차이**가 매우 크다. 1-3 파트에서 가졌던 의문처럼, 거리 기반 알고리즘인 k-NN이 이 스케일 불균형 때문에 거리를 왜곡해서 판단할 위험이 여전해 보인다. 2-2에서 배울 전처리가 이 고민을 해결해줄지 기대된다.
