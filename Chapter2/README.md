# [Chapter 2-1] 훈련 세트와 테스트 세트

### 1. 공부한 내용 요약
- **지도학습과 비지도학습**: 정답($y$)이 있는 데이터를 통해 상관관계를 찾는 지도학습과, 정답 없이 데이터 자체의 패턴을 찾는 비지도학습의 차이를 이해했다.
- **훈련 세트와 테스트 세트**: 모델의 일반화 성능을 정직하게 측정하기 위해 전체 데이터를 학습용 35게와 검증용 14개로 분리했다.
- **샘플링 편향(Sampling Bias)**: 무작위로 섞지 않고 순서대로 데이터를 나눌 경우, 특정 클래스가 한쪽에 몰려 모델 평가가 왜곡되는 현상을 확인했다.
- **NumPy 활용**: 파이썬 리스트를 넘파이 배열로 변환하고, 인덱스를 무작위로 섞어(np.random.shuffle) 편향 없는 데이터 세트를 구축했다.


### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **정확도 0.0이 준 교훈**: 처음 데이터를 나눌 때 단순히 슬라이싱만 했더니 점수가 0.0이 나왔다. 이는 모델의 결함이 아니라 데이터 배분의 문제였는데, 도미만 공부하고 빙어 시험을 치게 한 셈이라 통계학 전공자로서 데이터 분포의 중요성을 몸소 체험했다.
- **재현성을 위한 시드 고정**: 무작위 섞기를 할 때 np.random.seed(42)를 설정하는 법을 배웠다. 통계 분석에서 실험 결과의 재현은 필수적이므로, 이러한 랜덤성 제어 습관을 실무에서도 잘 챙겨야겠다고 생각했다.

- **인덱스 셔플링의 효율성**: 데이터 자체를 물리적으로 섞는 대신 인덱스 배열을 만들어 섞는 방식을 사용했다. 데이터가 매우 방대해질 경우 전체 데이터를 섞는 것보다 인덱스만 조절하는 게 메모리 측면에서 훨씬 유리할 것 같은데, 실제 대규모 데이터 처리 현장에서도 이 방식이 표준인지 궁금하다.
- **여전한 스케일 문제**: 데이터를 잘 섞어서 정확도는 다시 $1.0$이 나왔지만, 산점도를 그려보니 여전히 **x축(길이)과 y축(무게)의 단위 차이**가 매우 크다. 1-3 파트에서 가졌던 의문처럼, 거리 기반 알고리즘인 k-NN이 이 스케일 불균형 때문에 거리를 왜곡해서 판단할 위험이 여전해 보인다.
# [Chapter 2-2] 데이터 전처리

### 1. 공부한 내용 요약
- **데이터 결합**: np.column_stack()과 np.concatenate()를 사용하여 독립변수($X$)와 종속변수($y$)를 효율적으로 생성했다.
- **사이킷런으로 데이터 분리**: train_test_split()을 도입했으며, 특히 stratify 매개변수를 사용하여 클래스 비율이 훈련/테스트 세트에 골고루 배분되도록 설정했다.

- **스케일 문제 확인**: 길이와 무게의 단위 범위가 다를 때, 거리 기반 알고리즘인 k-NN이 특정 변수에만 의존하여 잘못된 예측(수상한 도미)을 수행함을 확인했다.
- **데이터 표준화(Standardization)**: 특성값에서 평균을 빼고 표준편차로 나누는 표준 점수($Z$-score) 공식을 적용하여 데이터의 스케일을 맞췄다.

- **NumPy 브로드캐스팅**: 넘파이 배열 간의 산술 연산이 행렬의 모든 행에 자동으로 적용되는 브로드캐스팅(Broadcasting) 기능을 활용하여 전처리를 수행했다.


### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **거리 기반 모델의 취약점**: 산점도상으로는 분명 도미 데이터에 가까웠던 샘플이 모델에게는 빙어 가깝게 인식되는 과정을 보며, 모델의 수리적 원리를 이해하지 못한 채 데이터를 넣는 것이 얼마나 위험한지 깨달았다. 통계학 전공자로서 시각화 결과만 믿기보다 수치적 스케일을 먼저 확인하는 습관을 가져야겠다고 생각했다.
- **테스트 세트 전처리의 주의점**: 훈련 세트의 평균과 표준편차를 그대로 사용하여 테스트 세트를 변환해야 한다는 점이 가장 인상적이었다. 테스트 세트만의 통계량을 사용한다면 훈련 시 사용한 기준과 달라지게 된다.
- **표준화와 정규화의 선택**: 이번 실습에서는 표준 점수를 사용했지만, 데이터의 범위가 0과 1 사이로 제한되는 최소-최대 정규화(Min-Max Scaling)가 유리한 상황은 언제일지 궁금해졌다. 데이터에 이상치가 많을 때는 표준 점수가 더 강건하다는 통계적 상식이 머신러닝 모델의 성능에 어떻게 직결되는지 더 공부하고 싶다.
- **브로드캐스팅의 활용**: 넘파이의 브로드캐스팅이 단순히 편리함을 넘어 대용량 데이터 연산에서 연산 속도를 어떻게 최적화하는지, 그리고 딥러닝 프레임워크(PyTorch 등)에서도 동일한 논리가 적용되는지 궁금하다.
