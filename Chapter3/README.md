# [Chapter 3-1] k-최근접 이웃 회귀

### 1. 공부한 내용 요약
- **k-최근접 이웃 회귀(k-NN Regression)**: 분류와 달리 이웃한 샘플들의 평균값을 계산하여 연속적인 수치(농어의 무게)를 예측하는 방식을 학습했다.
- **데이터 구조 변경**: 사이킷런 모델의 입력 요구사항에 맞춰 1차원 배열을 reshape(-1, 1)을 통해 2차원 배열로 변환하는 과정을 실습했다.
- **결정계수($R^2$)**: 회귀 모델의 성능 지표인 결정계수를 확인했으며, 이는 타겟의 평균 예측 대비 모델의 오차가 어느 정도인지 비율로 나타내는 지표임을 이해했다.
- **과대적합(Overfitting) vs 과소적합(Underfitting)**: 훈련 세트와 테스트 세트의 점수 차이를 비교하여 모델의 복잡도를 진단하는 법을 익혔다.

### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **결정계수의 한계**: 결정계수 점수만으로는 실제 무게와 몇 g이나 차이가 나는지 직관적으로 알기 어렵다. 이를 보완하기 위해 mean_absolute_error를 사용하여 모델이 평균적으로 약 19g 정도의 오차를 보인다는 것을 수치로 확인한 과정이 유익했다.
- **과소적합(Underfitting)의 발생 사유**: 실습에서 테스트 세트 점수가 훈련 세트보다 더 높게 나오는 기이한 현상이 발생했다. 이는 데이터셋의 크기가 너무 작거나, 모델이 지나치게 단순하여 훈련 세트의 패턴조차 충분히 학습하지 못했기 때문임을 알게 되었다.<img width="3999" height="1740" alt="image" src="https://github.com/user-attachments/assets/e9a9851a-f557-497c-9fda-7255b6598f55" />

- **하이퍼파라미터 튜닝**: 이웃의 개수($k$)를 5에서 3으로 줄였을 때 훈련 세트의 점수가 올라가며 과소적합이 해결되는 것을 보았다. 통계학에서의 **편향-분산 트레이드오프(Bias-Variance Tradeoff)** 개념이 모델의 $k$값 설정에 직접적인 영향을 미친다는 점이 인상적이었다.
- **k-NN 회귀의 근본적 한계**: k-NN 회귀는 주변 데이터를 평균 내는 방식이므로, 만약 훈련 데이터의 범위를 크게 벗어나는 새로운 데이터(예: 아주 거대한 농어)가 들어온다면 예측값이 고정되어 버리는 문제가 발생하지 않을까 하는 의구심이 든다. 이 부분은 다음 장에서 다룰 선형 회귀에서 어떻게 보완되는지 확인해보고 싶다.
