# [Chapter 3-1] k-최근접 이웃 회귀

### 1. 공부한 내용 요약
- **k-최근접 이웃 회귀(k-NN Regression)**: 분류와 달리 이웃한 샘플들의 평균값을 계산하여 연속적인 수치(농어의 무게)를 예측하는 방식을 학습했다.
- **데이터 구조 변경**: 사이킷런 모델의 입력 요구사항에 맞춰 1차원 배열을 reshape(-1, 1)을 통해 2차원 배열로 변환하는 과정을 실습했다.
- **결정계수($R^2$)**: 회귀 모델의 성능 지표인 결정계수를 확인했으며, 이는 타겟의 평균 예측 대비 모델의 오차가 어느 정도인지 비율로 나타내는 지표임을 이해했다.
- **과대적합(Overfitting) vs 과소적합(Underfitting)**: 훈련 세트와 테스트 세트의 점수 차이를 비교하여 모델의 복잡도를 진단하는 법을 익혔다.

### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **결정계수의 한계**: 결정계수 점수만으로는 실제 무게와 몇 g이나 차이가 나는지 직관적으로 알기 어렵다. 이를 보완하기 위해 mean_absolute_error를 사용하여 모델이 평균적으로 약 19g 정도의 오차를 보인다는 것을 수치로 확인한 과정이 유익했다.
- **과소적합(Underfitting)의 발생 사유**: 실습에서 테스트 세트 점수가 훈련 세트보다 더 높게 나오는 기이한 현상이 발생했다. 이는 데이터셋의 크기가 너무 작거나, 모델이 지나치게 단순하여 훈련 세트의 패턴조차 충분히 학습하지 못했기 때문임을 알게 되었다.<img width="3999" height="1740" alt="image" src="https://github.com/user-attachments/assets/e9a9851a-f557-497c-9fda-7255b6598f55" />

- **하이퍼파라미터 튜닝**: 이웃의 개수($k$)를 5에서 3으로 줄였을 때 훈련 세트의 점수가 올라가며 과소적합이 해결되는 것을 보았다. 통계학에서의 **편향-분산 트레이드오프(Bias-Variance Tradeoff)** 개념이 모델의 $k$값 설정에 직접적인 영향을 미친다는 점이 인상적이었다.
- **k-NN 회귀의 근본적 한계**: k-NN 회귀는 주변 데이터를 평균 내는 방식이므로, 만약 훈련 데이터의 범위를 크게 벗어나는 새로운 데이터(예: 아주 거대한 농어)가 들어온다면 예측값이 고정되어 버리는 문제가 발생하지 않을까 하는 의구심이 든다. 이 부분은 다음 장에서 다룰 선형 회귀에서 어떻게 보완되는지 확인해보고 싶다.

# [Chapter 3-2] 선형 회귀와 다항 회귀

### 1. 공부한 내용 요약
- **k-NN 회귀의 한계 확인**: k-NN 모델은 훈련 세트의 범위를 벗어나는 데이터(외측치)에 대해 새로운 예측을 하지 못하고 인접 샘플의 평균만을 반복 출력하는 한계를 확인했다.
- **선형 회귀(Linear Regression)**: 특성과 타겟 사이의 선형적인 관계를 가장 잘 나타내는 직선의 방정식을 찾는 알고리즘을 학습했다.
- **모델 파라미터(Model Parameter)**: 사이킷런의 LinearRegression 모델이 학습 후 저장하는 계수(coef_)와 절편(intercept_)의 의미를 파악했다.
- **다항 회귀(Polynomial Regression)**: 선형 모델의 단순함을 보완하기 위해 특성을 제곱한 항을 추가하여 데이터를 더 잘 설명하는 곡선 모델을 구축했다.

### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **외삽(Extrapolation)의 중요성**: 50cm 농어의 무게를 k-NN은 1,033g으로 고정 예측한 반면, 선형 회귀는 1,241g으로 추세에 맞게 예측하는 것을 보았다. 실제 연구실에서 마주할 데이터는 항상 훈련 범주 내에 있지 않으므로, 모델의 확장성이 얼마나 중요한지 깨달았다.
- **선형 모델의 비현실적 결과**: 선형 회귀 모델이 도출한 방정식($Weight = 15.7 \times Length - 709$)에 따르면, 길이가 짧은 농어의 무게가 음수로 나올 가능성이 존재한다. 이는 수리적으로는 최적의 직선일지 몰라도 생물학적/통계적 상식에는 어긋나는 결과임을 인지했다.
- **다항 회귀를 통한 성능 개선**: 길이를 제곱한 특성을 추가($Weight = 1.01 \times Length^2 - 21.6 \times Length + 116$)함으로써 선형 모델보다 높은 $R^2$ 점수($0.97$)를 얻었고, 음수 예측 문제도 상당 부분 해결됨을 확인했다.
- **특성 공학의 깊이**: 이번에는 수동으로 특성을 제곱하여 추가했지만, 만약 특성이 더 많아진다면 어떤 조합을 추가해야 할지 판단하는 통계적 기준(예: $p$-value를 통한 변수 선택법 등)이 머신러닝의 '특성 공학(Feature Engineering)'과 어떻게 연결되는지 더 탐구해보고 싶다.

# [Chapter 3-3] 특성 공학과 규제

### 1. 공부한 내용 요약
- **다중 회귀(Multiple Regression)**: 여러 개의 특성을 사용하여 타겟을 예측하는 모델을 구축했으며, 특성이 많아질수록 모델이 훈련 세트에 강력하게 동화됨을 확인했다.
- **특성 공학(Feature Engineering)**: PolynomialFeatures를 활용해 기존 특성들의 교차항과 제곱항을 생성하여 모델의 표현력을 높였다.
- **규제(Regularization)**: 모델이 훈련 세트에 과도하게 학습되지 않도록 계수의 크기를 제한하는 릿지(Ridge)와 라쏘(Lasso) 회귀를 실습했다.
- **하이퍼파라미터 튜닝**: alpha 매개변수를 조정하며 모델의 복잡도를 제어하고, 그래프를 통해 과대적합과 과소적합 사이의 최적점을 찾는 과정을 수행했다.

### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **차원의 저주와 과대적합**: 특성의 개수가 샘플 개수(42개)보다 많은 55개로 늘어났을 때, 훈련 세트의 $R^2$가 0.99인 반면 테스트 세트 점수가 음수($-144$)가 나오는 극단적인 과대적합 현상을 목격했다. 통계학적으로 자유도(Degree of Freedom)가 부족할 때 모델이 얼마나 불안정해질 수 있는지 깨달았다.
- **규제 전 표준화의 당위성**: 릿지와 라쏘 모델은 계수의 크기를 기반으로 작동하므로, 특성마다 스케일이 다르면 규제가 불공정하게 적용된다. 따라서 규제 적용 전 반드시 StandardScaler를 통해 데이터를 표준화해야 함을 인지했다.
- **릿지와 라쏘의 차이점**: 릿지는 모든 특성을 사용하면서 계수를 균형 있게 줄이는 반면, 라쏘는 불필요한 특성의 계수를 0으로 만들어버리는 성질을 확인했다. 실습 중 라쏘 모델에서 계수가 0이 된 특성이 40개나 발생한 것을 보고, 라쏘가 변수 선택(Variable Selection)의 효과를 가질 수 있음을 이해했다.
- **적정 규제 강도 탐색**: alpha 값을 로그 스케일($10^{-3} \sim 10^2$)로 변화시키며 시각화했을 때, 훈련 세트와 테스트 세트의 점수 차이가 가장 적은 구간이 최적의 모델임을 확인했다. 실제 연구 현장에서 대규모 변수를 다룰 때, 통계적 유의성 검정 외에도 이러한 검증 곡선(Validation Curve)을 통한 파라미터 결정이 실무적으로 매우 중요함을 느꼈다.
