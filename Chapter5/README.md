# [Chapter 5-1] 결정 트리 (Decision Tree)

### 1. 공부한 내용 요약
- **로지스틱 회귀의 설명력 한계**: 화이트 와인 분류 문제에서 로지스틱 회귀 모델의 계수를 분석했으나, 전문가에게 모델의 판단 근거를 직관적으로 설명하기 어렵다는 문제를 파악했다.
- **결정 트리 도입**: 질문을 통해 데이터를 나누는 `DecisionTreeClassifier`를 사용했으며, `plot_tree()`를 통해 모델의 결정 과정을 시각화했다.
- **지니 불순도(Gini Impurity)**: 노드 분할의 기준이 되는 지니 불순도의 개념을 익혔으며, 부모 노드와 자식 노드의 불순도 차이인 '정보 이득'이 최대가 되도록 트리가 성장함을 이해했다.
- **가지치기(Pruning)**: 트리가 무제한으로 깊어지면 과대적합(Overfitting)이 발생하므로, `max_depth` 매개변수를 조정하여 모델의 일반화 성능을 높였다.
- **특성 중요도**: 모델이 학습에 가장 기여도가 높다고 판단한 특성을 확인한 결과, 당도(Sugar)가 분류의 핵심 기준임을 데이터로 증명했다.

### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **전처리와 모델의 관계**: k-NN이나 로지스틱 회귀와 달리 결정 트리는 데이터의 스케일에 전혀 영향을 받지 않는다. 통계학 전공자로서 표준화 과정 없이도 동일한 성능을 낼 수 있다는 점이 매우 효율적이라고 느꼈으나, 만약 이상치(Outlier)가 극단적으로 많을 때도 이 안정성이 유지될지 의문이다.
- **불순도의 수리적 의미**: 지니 불순도($1 - \sum p_i^2$) 외에도 정보이론의 엔트로피(Entropy)를 불순도 기준으로 사용할 수 있음을 알게 되었다. 두 방식이 트리의 구조 형성에 실질적으로 어떤 성능 차이를 만드는지 수치적으로 더 분석해 보고 싶다.
- **설명력과 성능의 트레이드오프**: 결정 트리는 설명력이 뛰어나지만, 단일 트리는 과대적합에 매우 취약하다는 단점을 확인했다. 이를 보완하기 위해 여러 개의 트리를 사용하는 앙상블 학습(Random Forest 등)이 실무에서 왜 표준으로 자리 잡았는지 그 필요성을 체감하는 계기가 되었다.
- **변수 선택의 자동화**: 특성 중요도를 통해 모델이 스스로 유의미한 변수를 선별하는 과정을 보며, 앞서 배운 라쏘(Lasso) 회귀의 변수 선택 기능과 어떤 차이점이 있는지, 그리고 두 방식을 혼합하여 변수를 정제하는 것이 가능한지 궁금하다.

# [Chapter 5-2] 교차 검증과 그리드 서치

### 1. 공부한 내용 요약
- **검증 세트(Validation Set)**: 테스트 세트를 사용하지 않고 모델의 하이퍼파라미터를 튜닝하기 위해 훈련 데이터의 일부(20%)를 추가로 분리하여 활용했다.
- **교차 검증(Cross Validation)**: `cross_validate` 함수를 사용하여 훈련 세트를 여러 폴드(Fold)로 나누고 반복 검증함으로써 모델 성능 평가의 통계적 신뢰도를 높였다.
- **그리드 서치(Grid Search)**: `GridSearchCV`를 통해 하이퍼파라미터 탐색과 교차 검증을 동시에 수행했으며, 최적의 조합을 찾은 후 전체 훈련 세트로 자동 재학습하는 과정을 실습했다.
- **랜덤 서치(Random Search)**: 매개변수 값이 연속적이거나 탐색 범위가 넓을 때, `RandomizedSearchCV`와 `scipy.stats`의 확률 분포 객체를 활용하여 효율적으로 하이퍼파라미터를 최적화했다.

### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **테스트 세트 보존의 중요성**: 하이퍼파라미터 튜닝에 테스트 세트를 활용할 경우 모델이 해당 데이터에 과대적합되어 실전 성능이 왜곡될 수 있음을 이해했다. 이는 통계학에서 데이터의 독립성을 유지해야 하는 원칙과 일맥상통한다고 판단했다.
- **K-폴드 교차 검증의 비용 효율성**: 5-폴드 교차 검증 시 모델을 5번 훈련시켜야 하므로 데이터가 커질수록 연산 비용이 증가한다. 실무에서 데이터 크기와 검증의 정밀도 사이에서 최적의 K값을 결정하는 통계적 기준이 무엇인지 고민이 필요하다.
- **하이퍼파라미터 간의 상호작용**: `max_depth`와 `min_samples_split` 등을 동시에 튜닝할 때, 각 변수가 독립적으로 작용하지 않고 서로 영향을 주고받으며 최적점을 형성하는 과정을 목격했다. 단순한 격자 탐색(Grid)보다 효율적인 베이지안 최적화(Bayesian Optimization) 같은 기법이 실제 연구 현장에서 더 선호되는지 궁금하다.
- **랜덤 서치의 확률적 이점**: 파라미터 값을 직접 지정하는 그리드 서치보다 넓은 범위를 무작위로 훑는 랜덤 서치가 때로는 더 적은 시도로 더 좋은 성능을 낼 수 있다는 통계적 샘플링의 힘을 체감했다. 특히 정수형(`randint`)과 실수형(`uniform`) 분포를 적절히 섞어 쓰는 테크닉을 익혔다.

# [Chapter 5-3] 트리의 앙상블 (Ensemble Learning)

### 1. 공부한 내용 요약
- **랜덤 포레스트(Random Forest)**: 결정 트리를 랜덤하게 만들어 숲을 구성하는 알고리즘으로, 부트스트랩 샘플과 무작위 특성 선택을 통해 모델의 분산(Variance)을 감소시켰다.
- **엑스트라 트리(Extra Trees)**: 전체 훈련 세트를 사용하되 노드를 무작위로 분할하여 계산 속도를 높였으며, 랜덤 포레스트와 유사한 성능을 내면서도 과대적합 억제력이 있음을 확인했다.
- **그래디언트 부스팅(Gradient Boosting)**: 깊이가 얕은 트리를 연속적으로 추가하며 이전 트리의 오차를 보정하는 방식으로, 편향(Bias)을 효과적으로 줄이는 과정을 실습했다.
- **최신 앙상블 라이브러리**: 히스토그램 기반 알고리즘인 `HistGradientBoosting`과 더불어 실무에서 널리 쓰이는 외부 라이브러리인 `XGBoost`, `LightGBM`의 성능을 비교 분석했다.

### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **앙상블의 통계적 이점**: 단일 결정 트리가 가졌던 고질적인 과대적합 문제를 여러 트리의 평균(Bagging) 또는 순차적 보정(Boosting)을 통해 해결하는 과정이 인상적이었다. 이는 통계학에서 표본 평균의 표준오차가 줄어드는 원리와 유사하다고 판단했다.
- **OOB(Out-Of-Bag) 샘플의 활용**: 부트스트랩 추출 시 선택되지 않은 남은 데이터를 검증 세트처럼 활용하는 `oob_score_` 속성을 통해, 별도의 검증 세트 없이도 모델의 일반화 성능을 예측할 수 있다는 점이 매우 효율적이었다.
- **알고리즘별 특성 중요도 차이**: 랜덤 포레스트는 당도 외에도 도수, pH를 어느 정도 골고루 활용하는 반면, 단일 트리는 특정 변수에 의존도가 높았다. 앙상블 모델이 특성을 더 객관적으로 평가하여 데이터의 잠재적 패턴을 더 잘 포착함을 이해했다.
- **연산 효율성과 성능의 균형**: 그래디언트 부스팅은 성능은 좋으나 속도가 느리며, 히스토그램 기반 방식이나 LightGBM은 이를 획기적으로 개선했다. 실무 환경에서는 단순 정확도뿐만 아니라 학습 및 추론 속도라는 기회비용을 반드시 고려해야 함을 깨달았다.
- **하이퍼파라미터의 민감도**: 부스팅 모델에서 `learning_rate`와 `n_estimators` 사이의 상호작용이 모델 성능에 지대한 영향을 미친다. 앞서 배운 그리드 서치나 랜덤 서치를 이러한 복잡한 앙상블 모델에 적용할 때 연산량을 어떻게 통제할지 알아보려 한다
