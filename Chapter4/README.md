# [Chapter 4-1] 로지스틱 회귀와 확률 예측

### 1. 공부한 내용 요약
- **데이터 전처리**: Pandas를 이용해 CSV 데이터를 불러오고, StandardScaler를 활용하여 5개의 특성(Weight, Length, Diagonal, Height, Width)을 표준화했다.
- **k-NN의 확률 예측**: predict_proba() 메서드를 통해 k-NN 모델이 예측하는 확률을 확인했으며, 이웃의 비율에 따라 확률이 결정되는 이산적인 특성을 파악했다.
- **로지스틱 회귀(Logistic Regression)**: 선형 방정식을 이용해 분류를 수행하는 알고리즘을 학습했으며, 이진 분류와 다중 분류에서의 작동 원리를 실습했다.
- **시그모이드와 소프트맥스**: 이진 분류에서는 시그모이드 함수를, 7개의 종을 분류하는 다중 분류에서는 소프트맥스 함수를 사용하여 $z$값을 확률로 변환했다.

### 2. 분석 및 궁금한 점 (My Analysis & Questions)
- **k-NN 확률의 불연속성**: $k=3$일 때 확률이 0, 0.33, 0.67, 1.0으로만 제한되는 것을 보며, 표본의 개수가 적을 때 k-NN이 제공하는 확률 정보가 다소 거칠다는 점을 인지했다. 이는 통계학적 추정의 정밀도 관점에서 로지스틱 회귀보다 불리할 수 있음을 이해했다.
- **이진 분류의 확률 메커니즘**: decision_function을 통해 얻은 $z$값을 시그모이드 함수($expit$)에 대입했을 때, 모델이 출력하는 양성 클래스의 확률과 완벽히 일치함을 확인했다. 이를 통해 로지스틱 회귀가 내부적으로 선형 회귀와 동일한 수식을 사용하되, 출력층에서 비선형 변환을 거친다는 것을 수리적으로 검증했다.
- **다중 분류에서의 소프트맥스 활용**: 7개 생선 종의 $z$값들을 소프트맥스 함수에 통과시켜 합이 1이 되는 확률 분포를 도출했다. 통계학에서 다항 로지스틱 회귀를 다룰 때와 마찬가지로, 각 클래스 간의 상대적인 강도를 지수 함수를 통해 증폭시켜 명확한 분류 경계를 형성함을 확인했다.
- **규제 강도의 제어**: LogisticRegression 클래스의 매개변수 C를 통해 규제를 조절했다. 앞서 배운 릿지/라쏘의 alpha와 달리 C는 값이 작을수록 규제가 강해진다는 역수 관계를 유의해야 하며, 이번 실습에서 C=20을 설정하여 적절한 복잡도를 유지했다.
- **더 나아가며**: 현재는 고정된 데이터셋에서 로지스틱 회귀를 수행했지만, 만약 데이터가 실시간으로 추가되는 상황이라면 매번 모델을 새로 학습시켜야 하는 효율성 문제가 발생할 것 같다. 다음 장에서 배울 점진적인 학습 방법(확률적 경사 하강법)이 이 문제를 어떻게 해결할지 궁금하다.
