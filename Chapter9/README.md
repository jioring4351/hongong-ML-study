# [Chapter 9-1] 순차 데이터와 순환 신경망 기초

### 1. 주요 개념 및 용어 정리
- **순차 데이터 (Sequential Data)**: 데이터의 요소들이 특정 순서를 가지고 있으며, 그 순서가 어긋날 경우 정보의 의미가 훼손되는 데이터(예: 문장, 주가 데이터 등)를 이해했다.
- **순환 신경망 (RNN)**: 뉴런의 출력이 다시 입력으로 들어가는 순환 고리를 통해 시퀀스 데이터를 처리하는 인공 신경망 구조를 학습했다.
- **셀 (Cell)**: RNN에서 반복적으로 실행되는 핵심 연산 단위를 의미하며, 이전 단계의 정보를 유지하는 역할을 한다.
- **은닉 상태 (Hidden State)**: 셀이 출력하는 값이 다음 단계의 입력으로 다시 전달될 때의 상태값을 의미하며, 이를 통해 시퀀스의 맥락(Context)을 파악한다.

# [Chapter 9-2] 순환 신경망으로 IMDB 리뷰 분류하기

### 1. 공부한 내용 요약
- **IMDB 데이터셋 분석**: 영화 리뷰 텍스트 데이터 25,000개를 활용했으며, 가장 자주 등장하는 단어 500개로 사전을 한정하여 학습을 진행했다.
- **데이터 패딩(Padding)**: 각 리뷰의 길이를 100단어로 통일하기 위해 `pad_sequences()`를 사용했다. 정보 손실을 최소화하기 위해 문장 끝부분의 정보를 보존하는 'pre' 설정을 기본으로 적용했다.
- **표현 방식 비교**: 500차원의 희소 벡터를 만드는 원-핫 인코딩 방식과, 16차원의 밀집 벡터를 만드는 단어 임베딩 방식을 각각 모델에 적용하여 비교 실습했다.
- **RNN 모델 훈련**: `SimpleRNN` 층을 사용하여 순차 데이터를 처리하고, 검증 세트의 손실 곡선을 모니터링하며 과대적합 지점을 분석했다.

### 2. 분석 및 비판적 검토 (My Analysis & Questions)
- **원-핫 인코딩의 통계적 비효율성**: 단어 사전 크기가 500개일 때는 큰 문제가 없으나, 실무에서 수만 개의 단어를 다룰 경우 입력 데이터의 99% 이상이 0으로 채워지는 '희소성(Sparsity)' 문제가 심각해진다. 이는 불필요한 연산량을 늘리고 모델의 수렴을 방해하는 요인임을 인지했다.
- **임베딩 층의 효율성**: 단어 임베딩은 16차원만으로도 원-핫 인코딩보다 우수한 성능을 보였다. 이는 통계학에서 다차원 척도법(MDS)이나 주성분 분석(PCA)으로 정보를 압축하는 것과 유사하게, 고차원의 텍스트 정보를 저차원의 연속적인 공간으로 투영함으로써 모델의 학습 난이도를 낮춘 결과라고 판단했다.
- **SimpleRNN의 구조적 취약점**: 학습이 진행됨에 따라 훈련 점수는 높아지지만 검증 점수가 정체되는 현상을 목격했다. 이는 단순 RNN이 가진 '단기 기억'의 한계, 즉 문장의 앞부분 정보가 뒤로 갈수록 희미해지는 기울기 소실(Vanishing Gradient) 문제 때문일 가능성이 크다.
- **하이퍼파라미터의 영향**: 임베딩 벡터의 크기(16)나 단어 사전의 크기(500)가 성능에 미치는 영향이 매우 크다. 통계적으로 최적의 임베딩 차원을 결정하기 위한 명확한 기준보다는 경험적인 튜닝이 많이 요구된다는 점이 딥러닝 모델 설계의 난점임을 다시 확인했다.
