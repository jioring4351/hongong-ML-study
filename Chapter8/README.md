# [Chapter 8-1] 합성곱 신경망의 구성 요소

### 1. 주요 개념 및 용어 정리
- **합성곱 (Convolution)**: 입력 데이터 위에 필터를 슬라이딩하며 요소별 곱셈과 합을 반복하는 연산이다. 이미지의 공간적 구조를 유지하며 특징을 추출하는 데 핵심적인 역할을 한다.
- **필터와 커널**: 신경망이 학습해야 할 가중치 파라미터이다. 여러 개의 필터를 사용함으로써 하나의 이미지에서 색상, 선, 질감 등 다양한 특징을 동시에 추출할 수 있다.
- **특성 맵 (Feature Map)**: 합성곱 연산을 통해 만들어진 결과물이다. 필터가 찾고자 하는 특징이 이미지의 어느 위치에 있는지를 나타내는 활성화 지도라고 볼 수 있다.
- **패딩 (Padding)**: 입력 배열의 주위를 특정 값(0 등)으로 둘러싸는 작업이다. '세임 패딩(Same Padding)'을 통해 입력과 출력의 크기를 동일하게 유지함으로써 정보 손실을 방지했다.
- **풀링 (Pooling)**: 가중치 없이 특성 맵의 차원을 줄이는 다운샘플링 기법이다. 최대 풀링(Max Pooling)을 통해 이미지의 미세한 변화에 모델이 너무 민감하게 반응하지 않도록(불변성 확보) 조절했다.



### 2. 분석 및 비판적 검토 (My Analysis & Questions)
- **CNN의 공간적 이점**: 7장에서 배운 밀집층(Dense Layer)은 이미지를 1차원으로 펼치면서 픽셀 간의 공간적 관계를 파괴한다. 반면 CNN은 2차원 구조를 유지하며 근접 픽셀 간의 상관관계를 직접 학습한다는 점에서 통계적 효율성이 훨씬 높음을 인지했다.
- **비선형성의 도입**: 합성곱 연산 역시 선형 결합이므로, 각 연산 후에 ReLU와 같은 활성화 함수가 반드시 결합되어야 복잡한 패턴을 학습할 수 있음을 상기했다.
- **파라미터 공유의 경제성**: 모든 픽셀에 별도의 가중치를 두는 대신, 하나의 필터가 이미지 전체를 훑으며 가중치를 공유(Weight Sharing)하는 방식이 모델의 복잡도를 획기적으로 줄여준다는 점이 인상적이었다.
- **정보 손실에 대한 의문**: 풀링 과정에서 데이터의 절반 이상이 버려지게 되는데, 이것이 단순한 노이즈 제거를 넘어 중요한 세부 정보를 소실하게 만들 위험은 없는지 궁금하다. 최근에는 풀링 대신 스트라이드를 조절하여 차원을 줄이는 방식도 쓰인다는데, 두 방식의 통계적 성능 차이를 비교해 보고 싶다.

# [Chapter 8-2] 합성곱 신경망을 사용한 이미지 분류

### 1. 공부한 내용 요약
- **모델 설계**: `Conv2D` 레이어 2개와 `MaxPooling2D` 레이어 2개를 교차 배치하여 특성 맵을 생성하고, 마지막에 `Flatten`과 `Dense` 층을 연결하여 10가지 의류 카테고리를 분류했다.
- **하이퍼파라미터 설정**: 각 합성곱 층에서 `padding='same'`을 사용하여 입력 크기를 유지했으며, 3x3 크기의 커널(Kernel)을 사용했다.
- **자동화된 학습 제어**: `ModelCheckpoint`를 통해 검증 점수가 가장 높은 시점의 가중치를 자동 저장하고, `EarlyStopping`을 사용하여 불필요한 연산을 차단했다.
- **성능 평가**: 훈련 세트와 검증 세트의 손실 곡선을 비교 분석했으며, 최종적으로 테스트 데이터셋에서 약 92%의 일반화 정확도를 확보했다.



### 2. 분석 및 비판적 검토 (My Analysis & Questions)
- **공간 정보 보존의 수리적 이점**: 7장의 밀집층 모델(정확도 약 87%)에 비해 CNN이 약 5%p 이상의 성능 향상을 보였다. 이는 통계적으로 인접 픽셀 간의 상관관계를 필터 연산으로 직접 포착한 결과이며, 이미지 분류에서 공간적 구조(Spatial Structure)가 얼마나 결정적인 정보인지 수치로 증명했다.
- **파라미터 수의 효율성**: 합성곱 층은 가중치를 공유(Weight Sharing)하기 때문에 동일한 깊이의 밀집층보다 파라미터 수가 훨씬 적음에도 불구하고 더 강력한 특징 추출 능력을 보여주었다.
- **특성 맵 시각화의 한계**: 모델이 훈련되면서 각 층의 필터가 어떤 특징(가로선, 세로선 등)을 학습했는지 시각적으로 확인할 수 있었으나, 심층부로 갈수록 추출되는 특징이 인간이 직관적으로 이해하기 어려운 형태로 변하는 '블랙박스'적 특성을 확인했다.
- **계산 복잡도와 하드웨어**: 성능은 우수하지만 밀집층 대비 합성곱 연산은 훨씬 많은 행렬 연산을 요구한다. 실무 환경에서 모델의 정확도 향상을 위해 층을 무작위로 늘리기보다, 추론 속도(Latency)와 자원 소모량 사이의 경제적 균형을 고려하는 것이 중요함을 깨달았다.
- **데이터 증강(Data Augmentation)의 필요성**: 현재는 주어진 정적인 데이터만 사용했지만, 이미지가 약간 회전하거나 뒤집힌 경우에도 현재의 성능이 유지될지 의문이다. 향후 실제 서비스 환경에 적용한다면 다양한 변형을 포함한 데이터 증강 기법을 병행해야 더 강건한 모델이 될 것으로 판단했다.

# [Chapter 8-3] 합성곱 신경망의 시각화

### 1. 공부한 내용 요약
- **가중치 시각화**: `layers[0].get_weights()`를 통해 첫 번째 합성곱 층의 가중치를 추출하고 이미지로 시각화하여, 모델이 학습한 필터의 패턴을 분석했다.
- **함수형 API의 활용**: 케라스의 `Model` 클래스를 사용하여 모델의 입력과 특정 중간 층의 출력을 연결하는 새로운 모델을 정의하고, 이를 통해 내부 연산 결과를 추출했다.
- **특성 맵 분석**: 훈련된 필터들이 입력 이미지(예: 신발 등)의 윤곽선이나 특징적인 부위에 강하게 반응(활성화)하여 특성 맵을 형성하는 과정을 관찰했다.
- **심층 층의 시각화**: 첫 번째 층뿐만 아니라 더 깊은 층의 특성 맵도 확인했으며, 층이 깊어질수록 특징이 더 추상적이고 복잡한 형태로 변하는 것을 파악했다.

### 2. 분석 및 비판적 검토 (My Analysis & Questions)
- **필터의 통계적 유의성**: 훈련된 필터가 명확한 기하학적 패턴을 보인다는 것은 모델이 데이터의 변동성(Variance) 중 의미 있는 특징을 성공적으로 학습했음을 시사한다. 반면 훈련되지 않은 가중치가 균등 분포에 가까운 무작위성을 보이는 점은 학습 과정의 필연성을 증명한다.
- **해석 가능성(Interpretability)의 한계**: 첫 번째 층은 직관적으로 이해 가능하지만, 층이 깊어질수록 특성 맵은 인간이 해석하기 힘든 형태로 변한다. 이는 딥러닝의 고질적인 '블랙박스' 문제를 보여주며, 시각화만으로는 모델의 전체적인 의사결정 논리를 완벽히 설명하기 부족함을 인지했다.
- **가중치 분포 분석**: 가중치의 히스토그램을 그렸을 때 훈련 후의 가중치가 특정 범위에 몰려 있는 분포를 확인했다. 이는 모델이 중요한 특징에 가중치를 집중시키는 '선택적 학습'을 수행했음을 의미하며, 통계적으로는 매개변수의 효율적인 최적화가 이루어졌다고 볼 수 있다.
- **실무적 활용 가치**: 시각화는 모델 디버깅 시 유용하다. 만약 모델이 잘못된 예측을 한다면 특성 맵을 통해 모델이 데이터의 엉뚱한 부분(예: 배경 노이즈 등)에 집중하고 있지는 않은지 판단하는 근거로 활용할 수 있을 것으로 기대된다.
